# Shadow AI Watch Companion

**Sprint:** 55  
**Product:** Lumen  
**Status:** Active

---

## Purpose

**Shadow** is Lumen’s AI watch companion — a bounded context for guided watching, not a generic chat panel beside a video player.

Shadow reuses the existing video pipeline outputs (transcript, translation, conversation memory) as **interactive temporal memory**. It does **not** duplicate speech-to-text, translation generation, or final render processing.

Core concepts:

- **ShadowSession** — immutable aggregate with playback state and append-only interactions
- **WatchContext** — resolved transcript/translation context at a playback timestamp
- **ShadowInteraction** — question, answer, pause, or resume at a video timestamp

---

## Timeline context resolution

`CurrentSegmentResolver` finds the transcript segment at a timestamp (exact match or nearest midpoint).

`TimelineContextBuilder` builds a deterministic window (±2 segments) of nearby transcript and translation text.

`ShadowContextFactory` loads artifacts via existing repositories — no LLM in context resolution.

**Fallback:** if translation is missing for the target language, context uses transcript only.

## Backend command model

| Command | HTTP | Effect |
| ------- | ---- | ------ |
| Start session | `POST /api/videos/{videoId}/shadow/sessions` | Creates playing session |
| Ask question | `POST .../sessions/{sessionId}/ask` | Resolves context, calls chat provider, stores Q&A |
| Pause | `POST .../sessions/{sessionId}/pause` | Updates session state only |
| Resume | `POST .../sessions/{sessionId}/resume` | Updates session state only |
| Get context | `GET /api/videos/{videoId}/shadow/context?time=&language=` | Read-only context |

**Important:** pause/resume commands model session state. The **browser video element** controls actual playback. The backend never manipulates the player directly.

---

## Frontend watch experience

Route: `/video/:videoId/watch`

- Video player uses final render stream when available
- Debounced context fetch on `timeupdate` (400ms)
- Shadow conversation panel with text input (always available)
- Browser `SpeechRecognition` for voice questions when supported
- `speechSynthesis` for answer playback with text fallback
- **Multilingual voice (Sprint 56.5):** speaking language selector (Auto / EN / FR / DE), browser voice matching, answer language metadata from backend
- **Adaptive intelligence (Sprint 57):** when enabled, explanation style hints and voice language may follow learned preferences; manual voice selection always wins
- Transcript/translation panels highlight the current segment
- UI localized in English, French, and German

## Limitations (Sprint 55)

| Area | Limitation |
| ---- | ---------- |
| Session persistence | In-memory repository (resets on deploy) |
| Vocabulary panel | Placeholder UI; proactive tutor handles vocabulary checks (Sprint 56) |
| Speaker detection | Not available in transcript segments |
| Video source | Requires final render for playback; no source-file stream in UI |
| Voice | Browser APIs only; quality varies by browser; no server TTS yet |
| Multilingual voice | EN/FR/DE via browser voices + backend answer language resolution (Sprint 56.5) |
| Adaptive learning | Optional; off by default; requires user to enable at `/settings/learning` (Sprint 57) |

## Adaptive integration (Sprint 57)

When the user enables adaptive recommendations (`LearningProfile.adaptiveRecommendationsEnabled`):

- Shadow answer prompts may include a learned explanation style hint (short vs detailed)
- Voice language may default to a learned preference when the user has not set manual voice mode
- Proactive tutor challenge level may soften or increase based on skip/answer patterns

When adaptive is disabled or no profile exists, Sprint 55–56.5 behavior is unchanged.

See [ADAPTIVE_INTELLIGENCE_ENGINE.md](./ADAPTIVE_INTELLIGENCE_ENGINE.md).

---

## Why Shadow does not replace the video pipeline

Shadow **reads** transcript and translation artifacts produced by the pipeline. Asking “explain this sentence” uses segment text already generated by Faster-Whisper and translation providers.

Adding Shadow does not change `ProcessVideoHandler`, STT providers, or translation generators.
