# Stories

US-UI-0001

As a developer

I want reusable Button, Input, and Card components

So that Features stay visually consistent.

---

US-UI-0002

As a developer

I want Dropzone, Progress, and StatusChip components

So that upload and processing Features share the same patterns.

---

US-UI-0003

As a developer

I want Modal, Dialog, Toast, Spinner, EmptyState, and ErrorState

So that feedback patterns are unified.

---

US-UI-0004

As a developer

I want DashboardLayout and page shells

So that navigation structure exists before Features ship.

---

US-UI-0005

As a user

I want to see a Dashboard with import actions and an empty library state

So that I understand what History AI will offer.

---

US-UI-0006

As a developer

I want Storybook stories for every generic component

So that I can develop UI without the backend.
