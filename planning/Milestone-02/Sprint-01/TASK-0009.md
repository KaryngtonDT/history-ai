# TASK-0009 — Create Content API Endpoint

## Milestone

M2 — Content Ingestion

## Sprint

Sprint-01

## Objective

Expose the Create Content use case through a minimal REST API endpoint.

---

## Endpoint

POST /api/contents

---

## Request Body

```json
{
  "title": "The Roman Empire",
  "sourceType": "youtube_url"
}
```

---

## Success Response

HTTP 201

```json
{
  "id": "uuid"
}
```

---

## Error Response

HTTP 400

```json
{
  "error": "Invalid request"
}
```

---

## Requirements

Create:

- CreateContentController
- CreateContentRequest DTO
- CreateContentResponse DTO
- Functional tests

The controller must:

- decode JSON request
- validate required fields
- call CreateContentHandler
- return HTTP 201 with ContentId
- return HTTP 400 for invalid input

---

## Backend Structure

```text
backend/src/Presentation/Http/Controller/Content/
backend/src/Presentation/Http/Request/Content/
backend/src/Presentation/Http/Response/Content/
```

---

## Constraints

Do not add API Platform.

Do not add authentication.

Do not create upload logic.

Do not create MinIO integration.

Do not create queue logic.

Do not create AI logic.

Do not put business logic in the controller.

---

## Tests

Create functional tests for:

- successful content creation
- missing title
- invalid sourceType

---

## Acceptance Criteria

- POST /api/contents creates Content.
- Valid request returns 201.
- Invalid request returns 400.
- Content is persisted.
- Tests pass.
- Existing tests remain green.

---

## Commands

Run:

```bash
docker compose exec backend php bin/phpunit
```
