# TASK-0010 — List Contents API Endpoint

## Milestone

M2 — Content Ingestion

## Sprint

Sprint-01

## Objective

Expose a minimal REST API endpoint to list created Contents.

---

## Endpoint

GET /api/contents

---

## Success Response

HTTP 200

```json
[
  {
    "id": "uuid",
    "title": "The Roman Empire",
    "sourceType": "youtube_url",
    "status": "draft",
    "createdAt": "ISO-8601",
    "updatedAt": "ISO-8601"
  }
]
```

---

## Requirements

Create:

- ListContentsQuery
- ListContentsHandler
- ContentListItem DTO
- ListContentsController
- Functional tests

The controller must:

- call the ListContentsHandler
- return HTTP 200
- return an array of content items

---

## Backend Structure

Application:

```text
backend/src/Application/Content/Queries/
backend/src/Application/Content/Handlers/
backend/src/Application/Content/DTO/
```

Presentation:

```text
backend/src/Presentation/Http/Controller/Content/
backend/src/Presentation/Http/Response/Content/
```

---

## Constraints

Do not add pagination yet.

Do not add filters.

Do not add search.

Do not add authentication.

Do not expose Doctrine entities.

Do not put database logic in the controller.

Do not create frontend changes.

---

## Tests

Create tests for:

- empty content list returns []
- list contains created content
- response shape is stable

---

## Acceptance Criteria

- GET /api/contents returns 200.
- Empty database returns [].
- Created contents are listed.
- Response does not expose Doctrine internals.
- Tests pass.
- Existing tests remain green.

---

## Commands

Run:

```bash
docker compose exec backend php bin/phpunit
```
