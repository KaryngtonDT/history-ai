# TASK-0008 — Create Content Use Case

## Milestone

M2 — Content Ingestion

## Sprint

Sprint-01

## Objective

Implement the application use case responsible for creating a new Content.

No HTTP endpoint.

No upload.

No storage.

No queue.

---

## Business Context

The platform must be able to create a Content aggregate before any processing starts.

The use case orchestrates the domain and persistence layers.

---

## Requirements

Create:

* CreateContentCommand
* CreateContentHandler
* CreateContentResult DTO

The handler must:

* validate the command
* create a Content aggregate
* save it through the repository interface
* return the created ContentId

---

## Backend Structure

```text
backend/src/Application/Content/
  Commands/
    CreateContentCommand.php
  Handlers/
    CreateContentHandler.php
  DTO/
    CreateContentResult.php
```

---

## Constraints

Do not create HTTP controllers.

Do not create upload logic.

Do not create MinIO integration.

Do not create queue jobs.

Do not create events.

Do not create AI processing.

---

## Tests

Create unit tests for:

* valid creation
* empty title rejection
* repository save called exactly once

Mock the repository.

Do not use Doctrine.

---

## Acceptance Criteria

* Handler creates Content.
* Repository is called once.
* Result contains ContentId.
* Tests pass.
* Existing tests remain green.

---

## Definition of Done

The first application use case is implemented and fully tested.
