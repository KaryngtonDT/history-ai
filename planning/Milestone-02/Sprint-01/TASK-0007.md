# TASK-0007 — Content Domain Model

## Milestone

M2 — Content Ingestion

## Sprint

Sprint-01

## Objective

Create the minimal Content domain model.

No upload.

No storage.

No queue.

No AI.

---

## Business Context

A Content represents an educational source imported into the platform.

The Content is independent from its origin.

A Content may later come from:

- uploaded PDF
- uploaded audio
- uploaded video
- YouTube URL

This task only creates the backend domain model and persistence foundation.

---

## Requirements

Create:

- Content aggregate
- ContentId value object
- ContentTitle value object
- ContentSourceType enum
- ContentStatus enum
- Doctrine persistence entity
- Doctrine migration
- Repository interface
- Doctrine repository implementation
- Minimal unit tests

---

## Domain Rules

Content must have:

- id
- title
- sourceType
- status
- createdAt
- updatedAt

Allowed source types:

- upload_pdf
- upload_audio
- upload_video
- youtube_url

Allowed statuses:

- draft
- imported
- processing
- completed
- failed

Default status:

- draft

---

## Backend Structure

Use this structure:

```text
backend/src/Domain/Content/
backend/src/Application/Content/
backend/src/Infrastructure/Persistence/Doctrine/Content/
backend/tests/Unit/Domain/Content/
```

---

## Constraints

Do not create HTTP endpoints.

Do not create upload logic.

Do not create MinIO logic.

Do not create queue logic.

Do not create AI logic.

Do not create frontend changes.

Do not expose this model through API yet.

---

## Acceptance Criteria

- Content domain object can be created.
- Invalid empty title is rejected.
- Content has default status draft.
- Doctrine migration exists.
- Unit tests pass.
- Existing health test still passes.
- No business logic outside Domain.
