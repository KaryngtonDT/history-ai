> Un mentor répond. Un mentor exécutif **pilote un parcours**.

---

# Objective

Shadow orchestrates the full learning path: what to learn now, what to defer, what to revise, when to slow down or accelerate, when to skip content, and when to recommend a better resource.

Every decision is:

- deterministic ;
- explainable ;
- user-validatable ;
- reversible.

Shadow is **never autonomous**. It **proposes**; the user **approves**.

# Executive principles (hard constraints)

The engine must **never**:

- modify the user profile ;
- change preferences ;
- delete data ;
- change a goal ;
- act without a pending user decision.

It may **only propose** decisions with `Approve` / `Skip` / `Defer` / `Never suggest again`.

Example:

```text
Recommendation: Review Docker first
Reason: Kubernetes depends on Docker
Actions: Approve | Skip | Never suggest again
```

## P66-SLICE-02 — Executive engine

- `ExecutivePlanner`, `ExecutiveDecisionBuilder`
- `PriorityResolver`, `ReviewScheduler`

## P66-SLICE-03 — Executive Shadow (watch conversation)
