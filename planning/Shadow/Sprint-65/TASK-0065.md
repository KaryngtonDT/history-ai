# TASK-0065 — Shadow Mentor & Goal Engine (Sprint 65)

## Phase

**Phase 6 — Personal Mentor**

## Status

Planned — **do not implement until Sprint 64 is signed off in production and this task is explicitly approved.**

## Strategic note

Sprints 55–64 made Shadow **intelligent**. Sprint 65 makes Shadow **goal-directed**. This is the last major functional capability before platform hardening (S66) and public API (S67).

Slow down net-new surface area elsewhere; invest in mentor moat: goals, roadmaps, missions, explainability, user approval.

---

# Objective

Shadow becomes a **personal mentor**: it understands *why* the user learns, plans a transformation path, assigns missions, measures progress, and adapts — always deterministically and with user validation.

---

# Philosophy shift

**Before (S55–S64)**

```text
Question → Memory → Knowledge Graph → Teaching → Answer
```

**After (S65)**

```text
Career Goal → Mentor Engine → Knowledge Graph → Teaching → Learning Mission → Conversation
```

The user no longer follows videos. They follow a **transformation journey**.

# Slices

## P65-SLICE-01 — Goal domain

`Domain/ShadowGoals/`:

- `LearningGoal`, `CareerGoal`, `SkillGoal`
- `GoalCategory`, `GoalPriority`, `GoalStatus`
- `GoalMilestone`, `GoalConstraint`
- `ShadowGoalsRepositoryInterface`
- `Exception/InvalidShadowGoalException`

Categories: `career`, `language`, `programming`, `history`, `philosophy`, `university`, `certification`, `personal`, `custom`

Each goal: description, motivation, priority, deadline, target skills, required knowledge, success criteria, progress percent.

**Multi-goal (stretch P65-SLICE-09):** primary + secondary goals with orchestrated mission allocation.

---

## P65-SLICE-02 — Mentor engine

`Application/ShadowMentor/`:

- `GoalPlanner`, `GoalAnalyzer`, `LearningMissionBuilder`
- `GoalProgressCalculator`, `GoalRecommendationEngine`
- `MilestoneResolver`, `GoalExplanationBuilder`, `MentorAdvisor`, `RoadmapBuilder`

**Inputs:** Knowledge Graph (S64), Memory (S62), Relationship (S61), Teaching (S63), Adaptive Learning, user preferences.

**Output:** `MentorPlan` (roadmap, current mission, milestones, skills radar, estimated completion).

Deterministic. No model training. Every recommendation explainable.

---

## P65-SLICE-03 — Learning missions

New concept distinct from Teaching missions (S63) — mentor-scoped, goal-linked.

Mission fields: title, objective, duration, prerequisites, exercises, validation, unlocked concept.

Wire completion → `KnowledgeBuilder` + `TeachingBuilder` sync.

API: `POST /api/shadow/missions/{id}/complete`

---

## P65-SLICE-04 — Mentor dashboard (frontend)

Route: `/settings/shadow/mentor`

Tab in `ShadowSettingsPage` alongside identity / relationship / memory / teaching / knowledge.

Panels: current goal, roadmap, current mission, next milestone, ETA, skills progress.

Services: `frontend/src/services/shadowMentor/*` (Repository / Http / Mock / Service pattern).

Features: `frontend/src/features/shadowMentor/MentorCenter/`

i18n: EN / FR / DE (`shadowMentor.{en,fr,de}.ts`).

---

## P65-SLICE-05 — Mentor conversation (watch)

`MentorContextComposer` → `ShadowWatchPromptBuilder`

Examples:

- "This video is directly linked to your goal."
- "I recommend this other video instead."
- "We can skip this section."
- "You are ready for the next step."

**Goal Impact Preview (stretch P65-SLICE-10):** before watch/mission, show impact % per active goal.

Watch panel: `ShadowMentorPanel` (poll mentor state like Teaching/Knowledge panels).

---

## P65-SLICE-06 — Roadmap builder

Horizons: today → this week → this month → this quarter → final goal.

User-editable. Deterministic regeneration on goal or progress change.

API: `GET /api/shadow/roadmap`

---

## P65-SLICE-07 — Weekly goal review

Proposed review flow: bilan → progression → milestones → difficulties → recommendations → "Adapt plan?" (requires user approval).

No silent plan mutation.
