## Sprint rule

> **1 Sprint = 1 clear product objective.**

**Sprint 68 objective:** build the **Shadow Presence layer** — the shared architecture that future companions plug into.

**Not in this sprint:** Browser extension, IDE extension, Mobile app, Ambient voice, clipboard hooks, page analysis.

---

# Vision

Lumen is no longer only a web platform.

Shadow becomes a permanent learning companion capable of accompanying the user across surfaces (Lumen, Desktop, Browser, IDE, Mobile) while keeping **one memory, one identity, one conversation**.

Sprint 68 does **not** deliver all integrations.

It builds the **common architecture** so future companions reuse the same brain.

> **One Shadow. Multiple Presences.**

# Architecture principles

| Principle | Rule |
| --------- | ---- |
| **One Shadow** | One profile across web, desktop, and future surfaces |
| **One Memory** | All interactions feed Brain, Memory, Graph, Mentor, Executive, Teaching |
| **One Conversation** | Thread started on YouTube continues in Lumen or Desktop |
| **Privacy first** | No automatic capture; explicit, disableable, logged access |
| **Explainability** | Shadow always answers *why did I appear?* |

# API

| Method | Path | Purpose |
| ------ | ---- | ------- |
| GET | `/api/shadow/presence/session` | Current presence session |
| POST | `/api/shadow/presence/connect` | Register surface connection |
| POST | `/api/shadow/presence/disconnect` | End surface connection |
| GET | `/api/shadow/presence/context` | Unified context for surface |
| PUT | `/api/shadow/presence/preferences` | Update presence preferences |
| GET | `/api/shadow/presence/history` | Activity / access log |
| GET | `/api/shadow/presence/explain` | Explainability for last action |

OpenAPI required. No `POST …/action` in S68 — dispatch via existing Shadow handlers through Context Hub.

# Success criteria

1. Presence domain + Context Hub return unified context from existing Shadow engines (no new intelligence).
2. Conversation started on web resumes on desktop with same session id.
3. `/settings/shadow/presence` configures permissions and surfaces (future stubs OK).
4. `GET /api/shadow/presence/explain` returns deterministic *why* for last presence event.
5. Tauri app connects, syncs profile, opens Quick Launcher — no OS surveillance.
6. Full Docker validation gate green.
