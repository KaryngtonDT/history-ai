## Strategic note

This is not a "Desktop app" sprint.

It is the **foundational milestone** for Shadow Presence — the architecture that enables Desktop, Browser, IDE, Mobile, and Ambient companions to share one intelligence.

> **One intelligence. Multiple points of presence.**

The Desktop Companion (Tauri) is the **first surface**, not the only deliverable.

---

# Vision

Shadow no longer lives only in Lumen.

Shadow becomes an omnipresent companion that accompanies the user everywhere they learn, work, read, code, or watch content.

**Goal of Sprint 68:** create the **unique architecture** for coherent Shadow presence — not yet every platform.

---

# Objectives

Create foundations where Shadow can:

- be invoked from any registered surface
- share one context across surfaces
- maintain a continuous conversation
- enrich Second Brain regardless of entry point
- apply the same identity, memory, mentor, and pedagogy rules

**No duplicate intelligence. One source of truth.**

---

# Architecture principles

| Principle | Rule |
| --------- | ---- |
| Shadow is unique | One identity across all surfaces |
| Shared context | Every interaction feeds Brain, Memory, Graph, Mentor, Executive |
| Privacy first | No automatic clipboard, code, or document access |
| Explicit access | Every read is user-initiated, temporary, revocable |
| Discrete presence | Appears on demand or opt-in proactive recommendation only |
| Lightweight desktop | Open < 300 ms — Raycast / Spotlight class, not a Lumen clone |

# API

| Method | Path | Purpose |
| ------ | ---- | ------- |
| GET | `/api/shadow/presence/session` | Current presence session |
| POST | `/api/shadow/presence/connect` | Register surface connection |
| POST | `/api/shadow/presence/disconnect` | End surface connection |
| GET | `/api/shadow/presence/context` | Unified context for surface |
| POST | `/api/shadow/presence/action` | Dispatch user action |
| PUT | `/api/shadow/presence/preferences` | Update presence preferences |
| GET | `/api/shadow/presence/history` | Access history (explainability) |

# Success criteria

1. User opens Desktop Companion in < 300 ms and asks about a Second Brain concept.
2. Conversation started on desktop continues in Lumen web with same thread.
3. Every presence session records explainable *why* in history.
4. No surface accesses content without explicit user action.
5. Full Docker validation gate green.
