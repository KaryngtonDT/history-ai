## Problem statement

Today the UI can show:

```text
Speech-to-Text

✓ Faster Whisper
```

While the pipeline actually runs:

```text
STT_PROVIDER=deterministic
```

Configuration, runtime, Docker, and worker are not fully synchronized. **This must never happen again.**

## Vision

Lumen becomes an **AI Engine Platform** — not merely an app that calls external tools.

Principle:

> **Configured. Verified. Measured. Intelligent. Explainable.**

Capabilities:

* discover installed engines automatically ;
* verify real readiness ;
* manage configuration ;
* measure performance ;
* benchmark quality ;
* recommend the best engine ;
* launch the pipeline intelligently ;
* explain every decision.

## Deployment model

### Docker (application stack)

```text
Frontend · Backend · Worker · PostgreSQL · Redis
```

### Host (AI engines)

```text
Ollama · Faster Whisper · F5-TTS · OpenVoice · LatentSync · FFmpeg
```

### Persistent paths

```text
models/   — model weights (never in images)
storage/  — application data
```

Benefits: no multi-GB Docker rebuilds, no model duplication, better GPU access, simpler engine updates.

### Runtime philosophy

The Runtime must **not** care whether an engine runs in Docker, on Windows, on Linux, via Ollama, or as a Python process. It only asks:

```text
Speech Engine → READY?
```

# Pipeline selection modes

Each pipeline stage supports four modes:

| Mode | Behavior |
| ---- | -------- |
| **AUTO** | Shadow / platform selects best engine |
| **PROFILE** | Balanced, Quality, Fast, Low VRAM, etc. |
| **MANUAL** | User picks engine |
| **RULES** | Content-type rules (e.g. Podcasts → Parakeet, Technical courses → Faster Whisper) |
